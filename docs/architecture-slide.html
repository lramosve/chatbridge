ChatBridge Architecture

Three design decisions that define the platform

1

Iframe Sandboxing

Apps run in <iframe sandbox> without allow-same-origin. The browser enforces isolation — apps cannot access the parent DOM, cookies, or storage. External API calls go through a FETCH_REQUEST proxy.

2

Typed PostMessage Protocol

9 message types define the contract. TOOL_INVOKE and TOOL_RESULT for function calls. STATE_UPDATE for LLM context. APP_COMPLETE for signaling. FETCH_REQUEST/RESPONSE for API proxy. Apps that implement this protocol work automatically.

3

Plugin Tools as AI SDK Function Calls

Each plugin publishes a manifest declaring tools as JSON Schema. These become Vercel AI SDK tool() definitions. The LLM sees them alongside built-in tools and routes via function calling — no custom routing needed.
